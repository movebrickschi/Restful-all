# Restful-all 功能文档

## 概述

Restful-all 是一个 IntelliJ IDEA 插件，用于发现、导航和调试多种框架的 REST API 路由。

### 支持框架

| 框架 | 语言/文件类型 | 检测方式 |
|------|-------------|----------|
| Spring Boot | `.java`, `.kt` | `@RestController`, `@Controller`, `@RequestMapping`, `@GetMapping` 等 |
| NestJS | `.ts`, `.tsx` | `@Controller()`, `@Get()`, `@Post()` 等装饰器 |
| Express.js | `.ts`, `.tsx`, `.js`, `.jsx` | `router.get()`, `app.post()` 等链式调用 |
| FastAPI | `.py` | `@app.get()`, `@app.post()` 等装饰器 |
| Flask | `.py` | `@app.route("/path", methods=["GET"])` |

### 工具窗口标签页

| 标签页 | 功能说明 |
|--------|----------|
| 接口 | 路由列表浏览与 API 调试 |
| 请求历史 | 查看所有历史请求与响应记录 |
| 前置 URL | 配置各模块的基础 URL 前缀 |
| 全局参数 | 配置自动附加到所有请求的公共参数 |

---

## 快捷键

| 快捷键 | 功能 | 说明 |
|--------|------|------|
| `Ctrl+Alt+/` (Mac: `Cmd+Alt+/`) | 打开路由搜索弹窗 | 支持按路径、方法、类名、函数名模糊搜索 |
| `Ctrl+Shift+C` | 复制 API 路由路径 | 光标在方法体内任意位置即可使用，也可通过右键菜单触发 |

## 1. 路由扫描与发现

### 自动扫描

- 项目启动时自动执行首次全量扫描
- 文件变更（编辑、创建、移动、删除）时增量更新，无需手动刷新
- 自动过滤注释中的路由声明（支持 `//`、`/* */`、`#` 注释）

### 扫描策略

- 最大文件大小限制：512 KB
- 跳过目录：`node_modules`、`dist`、`build`、`.git`、`.gradle`、`.idea`、`target`、`__pycache__`、`.next`、`.nuxt`、`vendor`、`venv`
- 路由去重规则：按 `方法:路径:文件:行号` 去重
- 排序规则：按路径排序，路径相同时按方法名排序

### HTTP 方法与颜色

| 方法 | 颜色 |
|------|------|
| GET | 蓝色 `#61AFEF` |
| POST | 绿色 `#98C379` |
| PUT | 黄色 `#E5C07B` |
| DELETE | 红色 `#E06C75` |
| PATCH | 紫色 `#C678DD` |
| HEAD | 青色 `#56B6C2` |
| OPTIONS | 灰色 `#ABB2BF` |
| ALL | 橙色 `#D19A66` |

## 2. 路由搜索弹窗

通过 `Ctrl+Alt+/` 打开，支持快速搜索和跳转。

### 功能

- 模糊搜索：按路径、方法、类名、函数名过滤（150ms 防抖）
- 预填充：如果编辑器中有选中文本，自动作为初始搜索词
- 每条结果显示：方法（彩色加粗）、路径、类名#函数名、文件:行号、框架标签
- 最多显示 200 条匹配结果

### 弹窗内快捷键

| 按键 | 操作 |
|------|------|
| `Up` / `Down` | 上下选择 |
| `Enter` | 跳转到选中路由的源码位置 |
| `Escape` | 关闭弹窗 |
| `F5` | 刷新（重新扫描路由） |

---

## 3. 接口列表

工具窗口「接口」标签页，上方为路由列表，下方为 API 调试面板（可拖拽调整比例）。

### 路由列表

- 四列表格：方法、路径、调试按钮、跳转按钮
- 顶部搜索框支持模糊过滤（150ms 防抖）
- 搜索框内可用 `Up` / `Down` 键导航列表
- 状态栏显示接口总数或匹配数

### 交互操作

| 操作 | 效果 |
|------|------|
| 点击调试图标 | 将路由加载到调试面板 |
| 点击跳转图标 | 跳转到源码位置 |
| 双击任意行 | 跳转到源码位置 |
| 点击刷新按钮 | 后台重新扫描项目路由 |

---

## 4. API 调试面板

### 请求配置

- **方法选择**：下拉框支持 GET、POST、PUT、DELETE、PATCH、HEAD、OPTIONS
- **URL 输入**：点击路由时自动拼接前置 URL
- **发送按钮**：发送请求，请求中按钮禁用并显示"发送中..."

### 请求参数标签页

| 标签页 | 说明 |
|--------|------|
| Query | 查询参数键值对表格，支持启用/禁用单个参数 |
| Body | 请求体编辑，支持 6 种类型（见下） |
| Path | 路径参数，自动从 `{param}` 或 `:param` 提取并填充 |
| Headers | 请求头键值对表格 |
| Cookies | Cookie 键值对表格 |

### Body 类型

| 类型 | Content-Type | 说明 |
|------|-------------|------|
| none | - | 无请求体 |
| form-data | `multipart/form-data` | 支持文件上传，文件类型自动检测 MIME |
| x-www-form-urlencoded | `application/x-www-form-urlencoded` | URL 编码的键值对 |
| json | `application/json` | JSON 文本编辑器（默认） |
| xml | `application/xml` | XML 文本编辑器 |
| raw | - | 纯文本，不自动设置 Content-Type |

### 响应展示

- **状态信息**：状态码（彩色）、耗时(ms)、响应大小(B/KB)
  - 2xx 绿色、3xx 黄色、4xx/5xx 红色
- **响应标签页**：Body（自动格式化 JSON）、Headers（表格）、Cookies（表格）
- **格式选择**：JSON / Text / HTML / XML

### 全局参数合并

发送请求时自动合并全局参数：
- Query、Headers、Cookies：局部同名参数覆盖全局参数
- Body：局部 Body 为空时使用全局 Body 内容（仅 json/xml/raw 类型）

### HTTP 客户端

- 连接超时：30 秒
- 请求超时：30 秒
- 自动跟随重定向

## 5. 前置 URL

### 配置表格

| 列名 | 说明 |
|------|------|
| 模块 | 模块名称，留空表示默认 |
| 类型 | 「自动识别」或「手动指定」 |
| 服务器 | 服务器地址，默认 `127.0.0.1` |
| 端口 | 服务端口号，默认 `8080` |
| 项目路径 | context-path，如 `/admin`、`/api` |
| 预览 | 自动生成的完整 URL（蓝色斜体） |

### 多模块项目支持

模块识别规则：
- **Spring Boot**: 向上查找包含 `build.gradle` / `build.gradle.kts` / `pom.xml` 的目录
- **NestJS / Express**: 查找包含 `package.json` 的目录
- **单模块项目**: 使用项目根目录名

### 自动检测

点击「自动检测模块」按钮：
1. 从已扫描路由中提取所有模块名
2. 读取 `application.properties` 或 `application.yml` 获取端口和 context-path
3. 自动填充到配置表格

### 匹配优先级

1. 精确匹配模块名
2. 回退到模块名为空的默认条目
3. 如果只有一条配置，直接使用

### 使用示例

```
my-project/
├── user-service/          # 端口 8081，context-path: /user
├── order-service/         # 端口 8082，context-path: /order
└── gateway/               # 端口 8080
```

路由 `/api/users`（属于 user-service）自动拼接为：`http://127.0.0.1:8081/user/api/users`

## 6. 全局参数

### 子标签页

| 标签页 | 说明 | 操作按钮 |
|--------|------|----------|
| Query | 查询参数键值对 | 保存、清空 |
| Body | 全局请求体（文本编辑器） | 保存、清空、格式化 JSON |
| Headers | 请求头键值对 | 保存、清空 |
| Cookies | Cookie 键值对 | 保存、清空 |

### 优先级规则

- 局部同名参数覆盖全局参数
- 局部 Body 为空时，自动使用全局 Body（仅 json/xml/raw 类型）

### JSON 格式化

Body 标签页左上角提供格式化按钮（图标），可将压缩的 JSON 自动美化缩进。

## 7. 请求历史

### 界面布局

- **左侧**：按日期分组的树形视图，每条显示方法（彩色）、URL（截断80字符）、时间
- **右侧**：选中条目后展示请求/响应详情
  - 请求：Query、Body、Headers、Cookies
  - 响应：Body、Headers

### 操作

| 操作 | 说明 |
|------|------|
| 点击历史条目 | 右侧显示详情，同时加载到调试面板可重放 |
| 搜索 | 按 URL 或方法过滤（150ms 防抖） |
| 刷新 | 重新加载历史记录 |
| 清空 | 删除所有历史记录 |

### 存储

- 最多保存 **500** 条记录，超出自动淘汰最早的记录
- 持久化存储，重启 IDE 后可用

## 8. 复制路由路径

- **快捷键**：`Ctrl+Shift+C`
- **右键菜单**：编辑器右键菜单顶部「复制 API 路由路径」
- **使用条件**：光标位于路由声明行或其方法体内任意位置
- **匹配规则**：自动匹配光标行上方最近的路由声明
- **反馈**：复制成功后状态栏提示"已复制路由: /xxx/yyy"

## 9. 工具窗口行为

- **位置**：IDE 底部
- **互斥显示**：
  - 打开 Restful-all 时，自动隐藏其他所有可见的工具窗口
  - 打开其他工具窗口时，自动隐藏 Restful-all
- **工具栏按钮**：主工具栏提供「API 调试」按钮可快速打开

## 10. 数据持久化

所有配置数据通过 IntelliJ Platform 的 `PersistentStateComponent` 持久化，存储在项目目录下的 `.idea/restful-all.xml` 文件中。

- 关闭/重启 IDE 后配置不丢失
- 每个项目独立存储
- 建议将 `restful-all.xml` 加入 `.gitignore`

### 持久化内容

| 数据 | 说明 |
|------|------|
| 前置 URL 列表 | 所有模块的 URL 配置 |
| 全局参数 | Query、Headers、Cookies、Body |
| 请求历史 | 最多 500 条完整请求/响应记录 |
